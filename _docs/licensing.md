### Key points
- You may use, modify, and distribute the code for personal, commercial, and open-source projects.
- Keep copyright and license notices in derivative works.
- Include the `LICENSE` file when you redistribute or package the site (containers, binaries, tarballs).
- Provide prominent notice when you change files.
- The software is provided “as is” without warranties—see the full license for details.

### Attribution guidance
- Leave the `LICENSE` file intact.
- Optionally mention in your README or footer that the site is based on the TomeksDEV wiki template (Apache 2.0).
- If you fork and republish, keep the license text bundled with the distribution.
